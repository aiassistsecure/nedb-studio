The two-way console

Read and write, in plain English

Ask a question and the model compiles it to NQL and runs it on the live engine. Ask to add or change a record and it shows you the exact row it would write — editable, with regenerate and an explicit commit. Reads and writes, one console, nothing hidden.

query console
# you type:
"add a client Mark Evans, phone 555-0199, email mark@acme.co"

# the console proposes — editable before you commit:
write → clients

  name   Mark Evans
  phone  555-0199
  email  mark@acme.co

📤

Architecture

A thin client, by design

The studio talks to a running nedbd over a URL (NEDB_URL, env-first with a Settings override). The engine owns storage, the log, MVCC, time-travel, integrity, and durability. The studio renders, plans actions with the AiAssist gateway, and forwards reads and writes — no database logic lives in the studio.

Get started

Run it locally

  1. Start a NEDB server   pip install nedb-engine && nedbd
  2. Point the studio at it   NEDB_URL=http://127.0.0.1:7070 (in .env)
  3. Run the studio   npm install && npm run dev
  4. Describe an app → Generate → Deploy → ask & add in plain English.

Generation and natural-language actions run through the AiAssist gateway (your provider, your key, server-side only). Clone from GitHub.
